# Analysis of Algorithms

## Introduction

Analysis of Algorithms is the process of evaluating the efficiency of an algorithm in terms of time and memory usage. It helps in selecting the most suitable algorithm for solving a problem.

## Why is it Important?

* Compares different algorithms.
* Predicts performance for large inputs.
* Helps build efficient and scalable applications.
* Optimizes resource usage.

## Order of Growth

Order of Growth shows how the running time of an algorithm increases as the input size increases.

Example:

* O(1) – Constant Time
* O(log n) – Logarithmic Time
* O(n) – Linear Time
* O(n²) – Quadratic Time

## Asymptotic Analysis

Asymptotic Analysis measures algorithm performance for large input sizes.

### Cases

* **Best Case:** Minimum time required.
* **Average Case:** Expected time required.
* **Worst Case:** Maximum time required.

## Asymptotic Notations

* **Big O (O):** Upper bound of complexity.
* **Theta (Θ):** Exact complexity.
* **Omega (Ω):** Lower bound of complexity.

## Time Complexity

Time Complexity represents how execution time grows with input size.

Examples:

* Linear Search → O(n)
* Binary Search → O(log n)
* Bubble Sort → O(n²)

## Space Complexity

Space Complexity measures the memory used by an algorithm during execution.

## Analysis of Loops

* Single loop → O(n)
* Nested loops → O(n²)
* Loop dividing input by 2 → O(log n)

## Analysis of Recursion

Recursive algorithms are analyzed using recurrence relations.

Example:

* Binary Search → O(log n)
* Merge Sort → O(n log n)

## Amortized Analysis

Amortized Analysis calculates the average cost of operations over a sequence of operations.

Example:

* ArrayList insertion has amortized complexity O(1).

## Conclusion

Algorithm analysis helps in understanding the efficiency of programs and choosing the best solution based on time and space requirements.
